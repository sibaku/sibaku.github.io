Defining the camera position

When displaying a 3D scene, we need to think about how such an image is formed. The obvious model is our own vision, although having two eyes is a bit more complicated. So we make it simpler and go for a camera, with a single image sensor.

The first thing we need to answer is, how to place the camera in the 3D world and how to describe that mathematically.

A common way to describe a camera is by specifying three values:

  1. A\mathbf{A}A: The position of the camera in the world
  2. C\mathbf{C}C: The point the camera is pointing towards ("looking at")
  3. u\mathbf{u}u: The "up"-vector, which describes which direction is the "up" direction in our model. Usually, the yyy-axis is chosen

The basic idea is to compute the coordinates of each object vertex in the coordinate system described by our camera. This involves two steps. The first step is to make our camera the new origin of the coordinates. This is actually pretty easy and we already covered the operation before: A translation by −A-\mathbf{A}−A: T(−A)\mathbf{T}(-\mathbf{A})T(−A). This will move A\mathbf{A}A to the origin and everything else will move relative to that.

Now we need to compute the coordinates of the moved points. This might sound hard, but it is actually pretty intuitive. Let's say you have a coordinage system drawn out on a sheet of paper. You now place a point somewhere on that paper. How do you find the xxx and yyy coordinates? You draw a line perpendicular to the axis such that it goes through the point. This line of course also intersects the axis you draw it perpendicular too. To get the coordinate, you measure how far away that intersection is from the origin.

This is exactly the projection of the position vector of the point onto the axis (you "drop the position straight down onto the axis")! How do we calculate this projection? If our axis vector has a length of 111, you may recall that the length of the projection is exactly the dot product of the vector and the axis!

So, given that we know the camera axes xw,yw,zw\mathbf{x}^w,\mathbf{y}^w,\mathbf{z}^wxw,yw,zw in world space (the camera is placed in the world), we can find the camera coordinates of a point also in the world system pw\mathbf{p}^wpw as:

pxc=pw⋅xwpyc=pw⋅ywpzc=pw⋅zw\begin{align*} \mathbf{p}^c_x &= \mathbf{p}^w \cdot \mathbf{x}^w \\ \mathbf{p}^c_y &= \mathbf{p}^w \cdot \mathbf{y}^w \\ \mathbf{p}^c_z &= \mathbf{p}^w \cdot \mathbf{z}^w \end{align*} pxc​pyc​pzc​​=pw⋅xw=pw⋅yw=pw⋅zw​

Now, if you recall how matrix multiplication works, this can be simply written as:

pc=((xw)T(yw)T(zw)T)pw=Rcpw\begin{align*} \mathbf{p}^c &= \begin{pmatrix}(\mathbf{x}^w)^T \\ (\mathbf{y}^w)^T \\ (\mathbf{z}^w)^T\end{pmatrix} \mathbf{p}^w \\ &= \mathbf{R_c} \mathbf{p}^w \end{align*} pc​=​(xw)T(yw)T(zw)T​​pw=Rc​pw​

Note: If our axes are normalized and perpendicular and form a right-handed system, the matrix Rc\mathbf{R}_cRc​ will be a rotation matrix, satisfying RcRcT=I\mathbf{R}_c \mathbf{R}_c^T = \mathbf{I}Rc​RcT​=I.

Also recall that we used matrices to transform the coordinates of points before, even if it was in 2D. To compose multiple such transformations we just used matrix multiplication and we will do it this time again, we just need to once again make a 4×44\times 44×4 matrix from Rc\mathbf{R}_cRc​ by putting it in the upper left part of a 4×44\times 44×4 identity matrix. With that we define the View matrix V\mathbf{V}V as:

V=RcT(−A) \mathbf{V} = \mathbf{R}_c \mathbf{T}(-\mathbf{A}) V=Rc​T(−A)

This matrix will transform a point in the world's coordinate system into one in the camera's system!

As this does not change per model, we can for example put that into the uniform data as a constant.

What is missing is how we can compute the xw,yw,zw\mathbf{x}^w,\mathbf{y}^w,\mathbf{z}^wxw,yw,zw axes.

For that we think about how our screen coordinates should work. We want the xxx-axis to point to the right and the yyy axis to point up, such that we can align it with the screen where the origin is the lower left.

Note: This is the convention used in the OpenGL API, but others, such as Vulkan, are different and might have the yyy axis point down. This isn't a big issues, you just need to be careful what definition is used.

If we use the "right-up" definition and want to stay with a right handed coordinate system, the zzz axis has to point "out" of the screen, so the other direction of our viewing direction. And this is where we start.

The camera looks from A\mathbf{A}A to the point C\mathbf{C}C, so the normalized view direction is C−A∣∣C−A∣∣\frac{\mathbf{C}- \mathbf{A}}{||\mathbf{C}-\mathbf{A}||}∣∣C−A∣∣C−A​. As mentioned, the zzz-axis points in the other direction, thus have:

z=A−C∣∣A−C∣∣\mathbf{z} = \frac{\mathbf{A}- \mathbf{C}}{||\mathbf{A}-\mathbf{C}||} z=∣∣A−C∣∣A−C​

Now we want to compute the "right" axis: xxx. For that we use the up direction u\mathbf{u}u. We can think about it as a first guess on where the final yyy-axis will be. To get xxx from yyy and zzz, you compute y×z\mathbf{y} \times \mathbf{z}y×z. Using our up vector instead and normalizing we get our second axis:

x=u×z∣∣u×z∣∣\mathbf{x} = \frac{\mathbf{u} \times \mathbf{z}}{||\mathbf{u} \times \mathbf{z}||} x=∣∣u×z∣∣u×z​

By the properties of the cross product, x\mathbf{x}x and z\mathbf{z}z are perpendicular. We now get the final vector as another cross product:

y=z×x\mathbf{y} = \mathbf{z} \times \mathbf{x} y=z×x

Note, that we don't need to normalize, as the length of the cross product will be ∣∣z∣∣∣∣x∣∣sin⁡α=1∗1∗1=1||\mathbf{z} || || \mathbf{x} || \sin\alpha = 1 * 1 * 1 = 1∣∣z∣∣∣∣x∣∣sinα=1∗1∗1=1, since both vectors are normalized and perpendicular.

With that, we have found everything we need to define the position and orientation of a camera using the View matrix V\mathbf{V}V. We call the coordinate system defined this way the view space.

You can program this yourself or use the following function in code:

/**
 * Computes a 4x4 view matrix for 3D space
 *
 * @param {AbstractMat} eye - The camera center
 * @param {AbstractMat} center - The point to look at
 * @param {AbstractMat} up - The up vector
 * @returns {Mat} The view matrix
 */
jsm.lookAt(eye, center, up);

// Example
// The camera is located at (-0.5,0,0) and
//  looks to the origin (0,0,0) with (0,1,0)
//  being the world's up direction
const V = jsm.lookAt(
  vec3(-0.5,0,0), vec3(0,0,0), vec3(0,1,0)
);
