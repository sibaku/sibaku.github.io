Geodesic Raytracer

This is an example of a raytracer tracing rays through curved space, meaning some regions take longer for the ray to cross than others, despite looking "the same" in our normal 3D space. You can think of it similarly to gravitational effects bending space! A curve tracing out the shortest path through some space is called a geodesic. In uncurved space this is just a normal straight line.

The Shadertoy demo can be found here: https://www.shadertoy.com/view/fdBGDz

The stretching of space is given as a function f⁡(x,y,z)\operatorname{f}(x,y,z)f(x,y,z). This function can be defined in code and uses my simple GLSL library glsl-autodiff. The metric is then computed as a graph of that manifold. The shader performs a numerical integration of the ray according to local geodesics. In this case, where the surface is defined as as point

r=(xyzf⁡(x,y,z))\mathbf{r} = \begin{pmatrix} x\\ y\\ z\\ \operatorname{f}(x,y,z)\end{pmatrix} r=​xyzf(x,y,z)​​

In this case, the formulas can be simplified a bit, which is done in code, as otherwise it would be too heavy to compute (even now, it might lag depending on your used GPU).

Here is a quick explanation of how we arrive at the formula used in the shader. It basically follows the above Wikipedia article and the one on Christoffel symbols. Personally, I can highly recommend the following playlist about tensor calculus and the corresponding book Introduction to Tensor Analysis and the Calculus of Moving Surfaces. This really helped me see both how cool it is to write stuff as tensors and in tensor notation and how much you can do with it.

The (covariant) metric tensor follows from this and basically defines, as the name says, a kind of metric on some space, which allows you to measure distances. The metric tensor can be indexed by two indices gijg_{ij}gij​ and can be defined using the base vectors as:

gij=ei⋅ej g_{ij} = \mathbf{e}_i \cdot \mathbf{e}_j gij​=ei​⋅ej​

So it basically measures the length and angles between the base vectors. So it kinda makes sense, that we use it to find the shortest path. Even if omitted in the notation, this may change for each point you evaluate it at. In the following, there will be a a bit of the Einstein summation convention. Simply said, diagonally aligned indices with the same name imply a summation, so aibi=a1b1+a2b2+a3b3a^i b_i = a^1b_1 + a^2b_2 + a^3b_3aibi​=a1b1​+a2b2​+a3b3​. As we are in 3D, the summation is from 1 to 3 (math convention, so we start at 1).

With that we have:

d2ckdt2+Γijkdcidtdckdt=0 \frac{d^2 c^k}{dt^2} + \Gamma_{ij}^k \frac{dc^i}{dt}\frac{dc^k}{dt} = 0 dt2d2ck​+Γijk​dtdci​dtdck​=0

We can directly solve for the acceleration:

d2ckdt2=−Γijkdcidtdckdt \frac{d^2 c^k}{dt^2} = - \Gamma_{ij}^k \frac{dc^i}{dt}\frac{dc^k}{dt} dt2d2ck​=−Γijk​dtdci​dtdck​

We can use the ray's current velocity (dcdt\frac{d\mathbf{c}}{dt}dtdc​) as initial values in the above equation to compute the acceleration and from that the new velocity and position.
