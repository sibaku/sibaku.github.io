Camera Ray Generation

You may come into the situation, that you need to generate rays through a pixel. This could be for clicking on the screen to select an object or for a ray tracing application. There are of course other articles about this, but I feel they overcomplicate it. Here, I want to show a pretty simple way to generate rays, given a projection and a view matrix typically used in OpenGL and other Frameworks/APIs. I won't go into the details of these matrices, as they are covered in detail in many places, such as Songho

If you are only interested in the code, here is a glsl example:

// This assumes the pixel position px to be in [0,1],
// which can be done by (x+0.5)/w or (y+0.5)/h (or h-y +0.5 for screens
// with top left origin) to sample pixel centers
vec3 createRay(vec2 px, mat4 PInv, mat4 VInv)
{
  
    // convert pixel to NDS
    // [0,1] -> [-1,1]
    vec2 pxNDS = px*2. - 1.;

    // choose an arbitrary point in the viewing volume
    // z = -1 equals a point on the near plane, i.e. the screen
    vec3 pointNDS = vec3(pxNDS, -1.);

    // as this is in homogenous space, add the last homogenous coordinate
    vec4 pointNDSH = vec4(pointNDS, 1.0);
    // transform by inverse projection to get the point in view space
    vec4 dirEye = PInv * pointNDSH;

    // since the camera is at the origin in view space by definition,
    // the current point is already the correct direction 
    // (dir(0,P) = P - 0 = P as a direction, an infinite point,
    // the homogenous component becomes 0 the scaling done by the 
    // w-division is not of interest, as the direction in xyz will 
    // stay the same and we can just normalize it later
    dirEye.w = 0.;

    // compute world ray direction by multiplying the inverse view matrix
    vec3 dirWorld = (VInv * dirEye).xyz;

    // now normalize direction
    return normalize(dirWorld); 
}

The basic idea is very simple. We reverse the transformation pipeline. We usually want our rays to be in world-space, so that's why we need the view matrix. Just as a reminder:

View matrix V\mathbf{V}V - Transforms world coordinates to view coordinates

Projection matrix P\mathbf{P}P - Transforms view coordinates to clip coordinates

Division by the homogenous coordinate brings us from clip space to normalized device coordinates (NDC). The projection matrix encodes a viewing frustum, a truncated pyramid, with a near plane and a far plane. The frustum will be transformed in a cube [−1,1]3[-1,1]^3[−1,1]3 after division by the homogenous coordinate. The near plane (all values with zzz equal to −n-n−n) will be mapped to −1-1−1, while the far plane is mapped to 111.

We start with a point in NDC on the near plane (it could be somewhere else though). So we already know the zzz-coordinate: −1-1−1. Next are the xyxyxy-coordinates. NDC starts at the bottom left and goes from −1-1−1 to 111 and represents your screen. If your screen coordinate system also starts on the bottom left, we can work directly with the coordinates, otherwise we need to invert the yyy-axis as y=h−yinvertedy = h - y_{inverted}y=h−yinverted​, where hhh is the screen's height.

(xy)\begin{pmatrix} x & y \end{pmatrix}(x​y​) lies in [0,w−1]×[0,h−1][0,w-1]\times[0,h-1][0,w−1]×[0,h−1], where www is the screen's width. First we transform this to be in [0,1][0,1][0,1]. You probably want to sample the pixel centers, so we add 0.50.50.5 and divide by www and hhh respectively.

xscreen=(x+0.5)/wx_{\text{screen}} = (x+0.5)/w xscreen​=(x+0.5)/w

yscreen=(y+0.5)/hy_{\text{screen}} = (y+0.5)/h yscreen​=(y+0.5)/h or yscreen=(h−y+0.5)/h y_{\text{screen}} = (h-y+0.5)/h yscreen​=(h−y+0.5)/h for top left screens.

Next we change to NDC.

xNDC=xscreen∗2−1x_{\text{NDC}} = x_{\text{screen}}*2 -1 xNDC​=xscreen​∗2−1

yNCD=yscreen∗2−1y_{\text{NCD}} = y_{\text{screen}}*2 -1 yNCD​=yscreen​∗2−1

Both values are now in [−1,1][-1,1][−1,1]. Our 3D point in NDC is then

pNDC=(xNDCyNDC−1) \mathbf{p}_{\text{NDC}} = \begin{pmatrix} x_{\text{NDC}} \\ y_{\text{NDC}} \\ -1 \end{pmatrix}pNDC​=​xNDC​yNDC​−1​​

Next we make the point homogenous by adding a 111 as the last coordinate.

p^NDC=(xNDCyNDC−11) \mathbf{\hat{p}}_{\text{NDC}} = \begin{pmatrix} x_{\text{NDC}} \\ y_{\text{NDC}} \\ -1 \\ 1 \end{pmatrix}p^​NDC​=​xNDC​yNDC​−11​​

I use the hat to symbolize homogenous points. Next we go to view space. For that we use the inverse projection matrix.

p^View=P−1p^NDC \mathbf{\hat{p}}_{\text{View}} = \mathbf{P}^{-1}\mathbf{\hat{p}}_{\text{NDC}} p^​View​=P−1p^​NDC​

The result is a homogenous point in view space located on the near plane. Now for the two important things. First: In view space, the camera lies at the origin. We are interested in the direction from the camera to the point we just computed. Since the camera is in the origin, the line through the point is just written as t∗pView t * \mathbf{p}_{\text{View}} t∗pView​ with t t t being all real values. Now the second important thing: Division by the homogenous coordinate to get the real 3D point won't change the direction of the x,y,zx,y,zx,y,z ray through the origin. So the xyzxyzxyz coordinates of p^View \mathbf{\hat{p}}_{\text{View}}p^​View​ already represent the direction we care about.

Note: For an orthographic projection, the ray has the same direction for all points: Along the z-axis. You can use the point on the near-plane as the ray origin. There are many ways to handle this scenario. You could for example check, if the last row of the projection matrix only contains a 1 on the last entry and is 0 elsewhere. Or keep track of it in the program. You could also adjust the algorithm above to use two points: One on the near and one on the far plane and use these two points to generate a ray. This should work for both perspective and orthographic projections.

If you really need view space directions, you can stop here and use the first three entries of p^View \mathbf{\hat{p}}_{\text{View}}p^​View​ and normalize them.

To get world space directions, we have to transform from view to world space with the inverse view matrix. Since we are only interested in the direction, not a concrete point, we set the homogenous part of p^View \mathbf{\hat{p}}_{\text{View}}p^​View​ to zero.

q^View=(p^View,xp^View,yp^View,z0) \mathbf{\hat{q}}_{\text{View}} = \begin{pmatrix} \mathbf{\hat{p}}_{\text{View},x} \\ \mathbf{\hat{p}}_{\text{View},y} \\ \mathbf{\hat{p}}_{\text{View},z} \\ 0 \end{pmatrix}q^​View​=​p^​View,x​p^​View,y​p^​View,z​0​​

Then the transformation:

q^World=V−1q^View\mathbf{\hat{q}}_{\text{World}} = \mathbf{V}^{-1}\mathbf{\hat{q}}_{\text{View}} q^​World​=V−1q^​View​

The first three components now describe our world ray, so we just normalize them and are finished.
